### Arquitectura del workflow (8 nodos)

```
Recibir Imagen (Webhook POST /webhook/vinoteca/scan/analizar)
  ↓
Preparar Mensajes (Code — construye {model: gpt-4o, max_tokens: 600, messages})
  ↓
Analizar Vision (HTTP Request POST OpenAI /v1/chat/completions)
  — contentType: raw, body expression con JSON.stringify(...)
  — credential: cuenta OpenAI (predefinedCredentialType: openAiApi)
  ↓
Parsear Resultado (Code — limpia backticks, parsea JSON, normaliza nulls)
  ↓ (paralelo)
  ├── Generar Imagen Studio (HTTP Request POST OpenAI /v1/images/generations)
  │     — model: gpt-image-1, size: 1024x1536, quality: medium, output_format: jpeg
  │     — prompt construido con nombre+bodega+añada del vino
  │     — credential: cuenta OpenAI
  │     — ⚠️ body expression debe estar en modo = (naranja) — verificar en editor
  │     — ⚠️ eliminar header Authorization hardcodeado si existe
  │
  └── Buscar Imagen Fallback (HTTP Request GET serpapi.com/search.json)
        — api_key hardcodeada, engine: google_images, num: 1
        — q: expresión con nombre+bodega+añada+'wine bottle'
  ↓ (ambas ramas → Extraer URL Imagen)
Extraer URL Imagen (Code — prioriza b64 de IA, fallback URL de SerpAPI)
  ↓
Responder (Respond to Webhook)
```

### Lógica de "Extraer URL Imagen"
```js
const b64 = $('Generar Imagen Studio').first().json?.data?.[0]?.b64_json ?? null;
const serpUrl = $('Buscar Imagen Fallback').first().json?.images_results?.[0]?.original ?? null;
imagen_url = b64 ? 'data:image/jpeg;base64,' + b64 : serpUrl;
```

## Estado del frontend

### Flujo de escaneo (funciona end-to-end desde móvil con HTTP)
1. Foto frontal → `callScanAnalizar()` → n8n webhook
2. n8n devuelve `{nombre, bodega, anada, region, denominacion, uva, tipo, imagen_url}`
3. `imagen_url` puede ser `data:image/jpeg;base64,...` (IA) o URL externa (SerpAPI) o `null`
4. WineForm pre-rellena campos; `useEffect` sincroniza cuando llegan async
5. Al guardar: upload imagen a `wine-labels` → insert en `wines` de Supabase
6. Si upload falla (bucket error, CORS, etc.) → insert continúa sin imagen

### Bugs corregidos en esta sesión
- `theme.radii.sm` → `theme.radius.sm` en SyncIndicator (crash en bodega)
- `WineForm` no sincronizaba `initialData` cuando llegaba async → añadido `useEffect`
- Upload de imagen bloqueaba el guardado si fallaba → ahora en try/catch propio
- `crypto.randomUUID()` no disponible en Safari iOS → polyfill en `src/lib/uuid.ts`
