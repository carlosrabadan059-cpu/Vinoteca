## Lo que se hizo en esta sesión

### 1. Corrección de orientación 180° (retrato invertido)
- **Problema:** Al hacer foto con el móvil girado 180°, la imagen llegaba invertida.
- **Solución implementada:** Leer `window.orientation` en el momento exacto del shutter press (`handleCapture` en `CameraView.tsx`) y pasar el ángulo a `captureFrame`. Si `angle === 180`, el canvas rota la imagen antes de devolver el dataUrl.
- **Limitación iOS:** Safari no reporta 180° via ninguna API (`window.orientation` devuelve 0 siempre para portrait invertido). Solución pragmática: botón **"Girar"** en la barra de preview que aplica rotación 180° adicional antes de `onCapture`.

Archivos clave:
- [src/lib/captureSource.ts](../src/lib/captureSource.ts) — `captureFrameFromVideo` con rama `angle === 180`
- [src/components/ui/CameraView.tsx](../src/components/ui/CameraView.tsx) — `previewRotation` state, `handleRotate`, `handleConfirm` con canvas rotation

### 2. Eliminación completa del QR

**Diagnóstico del problema original:**
- ZXing (`@zxing/browser`) arrancaba antes de que el video tuviera frames (`readyState=0`)
- `srcObject` no estaba asignado cuando se ejecutaba el efecto QR
- Se intentaron fixes (guard `canplay`, combinar effects) pero ninguno funcionó en dispositivo real

**Decisión de negocio:** Los QR en etiquetas traseras de vino apuntan mayoritariamente al portal AECOC (`id.aecocescanqr.es`), no a datos útiles del vino. El feature no aporta valor real.

**Código eliminado de:**
- `CameraView.tsx`: `BrowserMultiFormatReader`, estados `SCANNING_QR`/`QR_FOUND`, props `enableQR`/`onQrDetected`, refs `qrActiveRef`/`readerRef`/`lastDetected`, funciones `startQrLoop`/`stopQrLoop`
- `Scan.tsx`: `callScanIdentificarQR`, `qrHandledRef`, `qrSessionRef`, `handleQrDetected`, y todas sus referencias en JSX

El componente `CameraView` ahora tiene una interfaz limpia sin QR:
```typescript
export interface CameraViewProps {
  source:    CaptureSource
  hint?:     string
  onCapture: (dataUrl: string) => void
  onCancel:  () => void
  onError:   (err: Error) => void
}
```

### 3. Problema de caché PWA (pendiente en el dispositivo del usuario)
- El service worker sirve el bundle viejo que aún muestra "Buscando QR…"
- El código en producción (Vercel) ya es correcto — el problema es solo en el dispositivo
- **Fix para el usuario:** En iPhone, Ajustes → Apps → Safari → Borrar historial y datos del sitio web. O mantener pulsado el botón de recarga en Safari → "Recargar sin contenido de caché"
