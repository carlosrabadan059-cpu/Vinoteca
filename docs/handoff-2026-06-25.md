# Handoff — 2026-06-25

## Sesión actual

Diagnóstico y corrección de dos bloqueos estructurales en el workflow n8n **"Vinoteca – Scan Analizar"** (`NMQZ4zhYw3RjTcLp`) que impedían completar el flujo cuando el usuario fotografiaba solo la etiqueta frontal (sin imagen trasera).

## Problema resuelto

**Síntoma:** La llamada a `callScanAnalizar(front, undefined)` desde la app nunca devolvía respuesta (timeout 60 s) cuando no había imagen trasera.

**Causa raíz:** Dos nodos `Merge` con estrategia `combineByPosition` quedaban bloqueados porque sus puertos de entrada nunca recibían datos:

| Nodo merge | Puerto bloqueado | Nodo NoOp desconectado |
|---|---|---|
| `Merge QR` | puerto 1 | `NoDo1` (rama `false` de `21 Tiene Trasera`) |
| `Merge Datos` | puerto 0 | `NoDo` (rama `false` de `Tiene Nombre`) |

**Fix aplicado:** Se añadieron dos conexiones vía `mcp__n8n-mcp__update_workflow`:
1. `NoDo1` → `Merge QR` (sourceIndex: 0, targetIndex: 1)
2. `NoDo` → `Merge Datos` (sourceIndex: 0, targetIndex: 0)

**Verificación:** Test execution `439011` con `back: null` completó con `lastNodeExecuted: "41 Responder"` y status `success`. Respuesta JSON correcta con datos del vino y `has_qr: false`.
