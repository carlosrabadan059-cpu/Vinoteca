## Lo que se implementó en V1.2

Ver detalles completos en `docs/roadmap.md` (sección V1.2 ✅).

Resumen de los 7 pasos:

1. **Migración Supabase** — 4 columnas nuevas en `tastings`: `es_consumo_rapido` (bool NOT NULL DEFAULT false), `botella_terminada` (bool NOT NULL DEFAULT false), `ocasion` (text), `lugar` (text). Aplicada manualmente por el usuario en el SQL Editor (el proyecto Vinoteca `xagsblgwvfitqkzjtwyc` no está accesible vía MCP).

2. **`src/types/index.ts`** — 4 campos nuevos en `Tasting`.

3. **`src/hooks/useTastings.ts`** — Defensive spread en `loadTastings()` para compatibilidad IDB backward; defaults en `createTasting()`.

4. **`src/pages/NuevaCata.tsx`** — `es_consumo_rapido: false` en la llamada a `createTasting()`.

5. **`src/components/wine/ConsumoQuickForm.tsx`** — Nuevo componente. Bottom sheet con campos: fecha, ocasión, lugar, botella terminada (checkbox). Llama a `createTasting` con `es_consumo_rapido: true`.

6. **`src/pages/WineDetail.tsx`** — Zona de acciones escalable (array `.map()`). Botón "Registrar consumo" con borde y texto dorado. `ConsumoQuickForm` integrado.  
   **`src/components/wine/TastingMiniCard.tsx`** — Badge `CATA` (dorado) / `CONSUMO` (gris). Preview adaptado: consumos muestran ocasión/lugar, catas muestran notas truncadas. Puntuación solo visible en catas.

7. **`src/components/wine/TastingForm.tsx`** — Campos opcionales `ocasion` y `lugar` añadidos al formulario de cata completa.

**`src/components/ui/Modal.tsx`** también fue actualizado:
- Animación bottom sheet: `translateY(100%) → translateY(0)` con `cubic-bezier(0.32, 0.72, 0, 1)` 300ms + fade backdrop
- Ancho: `calc(100% - 40px)`, `maxWidth: 480`, `borderRadius: 20px` en las 4 esquinas
- `marginBottom: 16` para separar del borde inferior
- `maxHeight: 85dvh`

## Decisiones de diseño tomadas en esta sesión

- Reutilizar tabla `tastings` con discriminador booleano `es_consumo_rapido` (no tabla separada)
- `precio_pagado` excluido de V1.2 — reservado para una futura feature de "compras" con modelo propio
- Sección "Mis catas" mantiene su nombre (no renombrada a "Historial" todavía)
- Badge texto `CATA`/`CONSUMO` (no emojis)
- El Modal ahora es un componente genérico con animación — cualquier nuevo modal del proyecto hereda automáticamente el bottom sheet
